sctcrpy.tl.define_clonotypes¶

sctcrpy.tl.define_clonotypes(adata, *, metric='alignment', key_added='clonotype', cutoff=2, strategy='any', chains='primary_only', inplace=True, resolution=1, n_iterations=5, n_jobs=None)¶

Define clonotypes based on cdr3 distance.

For now, uses primary TRA and TRB only.

Parameters
metric : {‘alignment’, ‘levenshtein’}Literal[‘alignment’, ‘levenshtein’] (default: 'alignment')

“alignment” - Calculate distance using pairwise sequence alignment

and BLOSUM62 matrix

”levenshtein” - Levenshtein edit distance

cutoff : intint (default: 2)

Two cells with a distance <= the cutoff will be connected. If cutoff = 0, the CDR3 sequences need to be identical. In this case, no alignment is performed.

strategy : {‘TRA’, ‘TRB’, ‘all’, ‘any’, ‘lenient’}Literal[‘TRA’, ‘TRB’, ‘all’, ‘any’, ‘lenient’] (default: 'any')

“TRA” - only consider TRA sequences “TRB” - only consider TRB sequences “all” - both TRA and TRB need to match “any” - either TRA or TRB need to match “lenient” - both TRA and TRB need to match, however it is tolerated if for a

given cell pair, no TRA or TRB sequence is available.

chains : {‘primary_only’, ‘all’}Literal[‘primary_only’, ‘all’] (default: 'primary_only')

Use only primary chains (“TRA_1”, “TRB_1”) or all four chains? When considering all four chains, at least one of them needs to match.

Return type

Tuple, NoneOptional[Tuple]
